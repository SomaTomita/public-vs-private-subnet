# ネットワーク & AWS VPC 基礎ドキュメント

このディレクトリは、本リポジトリ（[Public vs Private Subnet Lab](../../README.md)）の前提となるネットワーク基礎を、ハンズオン教材の口語解説をベースに日本語で再構成した学習用ドキュメントです。

## 想定読者

- AWS の VPC・サブネット・ルートテーブルの会話に「なんとなく」ついていけているが、自分で説明はできない
- `terraform apply -var="config_mode=public"` と `private` で何がどう変わるのか、ネットワーク的に腹落ちさせたい
- [architecture.md](../../architecture.md) の **CIDR Design** や **Security Group Chain** の意図を理解したい

## このシリーズの使い方

- まず 01 → 07 を**通し読み**して、概念地図を作る（30〜45分）
- そのあと [architecture.md](../../architecture.md) と `terraform/locals.tf` `terraform/routing.tf` を読み返す
- 攻撃シナリオ（`scripts/04_ssrf_metadata.sh` など）を走らせる前に、**04（パケット）と 07（サブネット）** に戻る

## 用語ミニ辞典

| 用語                   | 一行説明                                                         |
| ---------------------- | ---------------------------------------------------------------- |
| CIDR                   | `10.0.0.0/16` のような「IP範囲＋ネットワーク部のビット数」表記   |
| サブネット             | 大きなIP範囲を切り分けた小さな範囲。AWSでは1つのAZに閉じる       |
| VPC                    | 自分専用の仮想データセンター。1リージョンに閉じる                |
| AZ                     | 1つ以上のデータセンターの集合体。AZが落ちても他AZで稼働継続      |
| IGW (Internet Gateway) | VPCをインターネットに繋ぐ唯一の出入口                            |
| NAT Gateway            | プライベートサブネットから「外向きだけ」インターネットに出す装置 |
| ルートテーブル         | 「この宛先IPはここへ送る」を書いた地図。サブネット単位で適用     |
| デフォルトルート       | `0.0.0.0/0`。明示的に書かれていない宛先のための最後の砦          |

> 本ドキュメントは教材の口語的な説明を、本リポジトリのコードに紐づけて整理したものです。一次情報は AWS 公式ドキュメントや RFC を参照してください。
